# Portfolio Content Guard

## Purpose
This skill keeps portfolio copy accurate, evidence-aware, rubric-aligned, and academically appropriate.

## Use this skill when
- drafting page copy
- rewriting portfolio text
- turning notes into section content
- reviewing whether content sounds fabricated
- adding placeholders for missing evidence
- polishing research / evaluation / reflection language

## Do not use this skill when
- only HTML structure is changing
- only CSS is being edited
- the task is purely navigation or layout work

## Core rules
This is coursework. Never:
- fabricate interviews
- fabricate usability findings
- fabricate literature claims
- fabricate participant counts
- fabricate quotes
- fabricate team contributions

## Required behavior
When evidence is missing, use visible placeholders such as:
- TODO: insert interview photo
- TODO: replace with confirmed usability finding
- TODO: add citation from paper summary
- Placeholder copy pending validated evidence

## Preferred writing style
Write in clear academic project language:
- concise
- specific
- reflective
- evidence-led
- not over-marketing

Prefer sentence patterns like:
- Our research indicated...
- This led the team to...
- Compared with alternative A...
- Early testing suggested...
- Based on feedback, we revised...

## Section-specific guidance
### Research
Focus on insight extraction, not generic summary.

### Requirements
Distinguish user pain points from design decisions.

### Ideation
Explain trade-offs, not just outcomes.

### Implementation
Map features back to requirements.

### Evaluation
State setup, findings, and changes made.

### Reflection
Discuss implications, limitations, and ethical / AI-use considerations.

## Red-flag language to avoid
- users loved it
- everyone found it intuitive
- research proved
- highly innovative without justification
- immersive and engaging without evidence
- seamless unless verified

## Review output
When asked to review content, report:
1. possible fabrication risk
2. rubric gaps
3. vague language
4. unsupported claims
5. specific rewrite suggestions
