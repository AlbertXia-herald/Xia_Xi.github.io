# Portfolio Rubric Review

## Purpose
This skill reviews the portfolio website against coursework expectations and identifies missing or weak areas before submission.

## Use this skill when
- the user asks for a portfolio completeness review
- the user asks what is missing for a high score
- the user wants a page-by-page audit
- the user wants pre-submission checking
- the user wants TODOs prioritized

## Do not use this skill when
- implementing UI from scratch
- doing a one-line text edit
- fixing isolated CSS bugs

## Review dimensions
Check the portfolio for:
- motivation clarity
- research gap evidence
- persona and stakeholder clarity
- journey map / pain points
- requirements clarity
- alternatives comparison
- low-fi / high-fi continuity
- system architecture explanation
- feature-to-requirement mapping
- usability testing evidence
- before-after iteration evidence
- team contributions
- references and AI disclosure
- navigation clarity
- visual consistency
- accessibility basics

## Severity levels
Classify issues as:
- Critical: blocks coursework alignment
- Major: likely weakens mark significantly
- Minor: polish issue
- Nice-to-have: optional improvement

## Output format
Return:
1. strengths
2. missing items
3. weak items
4. page-by-page action list
5. top 5 priorities before submission

## Special coursework note
Prefer identifying evidence gaps over purely visual critiques.
A beautiful site with weak process evidence is not enough.
