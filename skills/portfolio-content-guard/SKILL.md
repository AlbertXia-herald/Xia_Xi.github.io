# Portfolio IA Planner

## Purpose
This skill helps Codex plan the structure of the Maple Bridge process portfolio so it matches coursework expectations and remains easy to navigate.

## Use this skill when
- the user asks for sitemap planning
- the user wants page breakdowns
- the user wants section structure for a new page
- the user wants to convert a simple portfolio into a rubric-aligned evidence site
- the user wants homepage and subpage responsibilities clarified

## Do not use this skill when
- the user only wants a color change
- the user only wants a bug fix
- the user only wants to edit wording in one paragraph
- the task is purely front-end implementation with already-decided structure

## Planning rules
Always plan around the process-portfolio goal, not a product landing page.

Default page map:
- Home
- Motivation & Research
- User Requirements
- Ideation & Alternatives
- Technical Implementation
- Evaluation & Reflection
- Team Contributions
- References & AI Disclosure

## Output format
When using this skill, produce:
1. page map,
2. purpose of each page,
3. key sections for each page,
4. content dependencies,
5. suggested internal links,
6. implementation notes if relevant.

## IA principles
- Home should be a hub, not a content dump.
- Subpages should each have a single dominant purpose.
- Long evidence pages should have section anchors.
- Navigation labels should match coursework language closely.
- Repeated content should be minimized.
- Team page should not replace process pages.

## Quality bar
A good IA plan should make it obvious:
- what the project is,
- why it exists,
- how requirements were derived,
- what alternatives were compared,
- how the system was implemented,
- how it was evaluated,
- who contributed what.

## Special note
If the current site is too shallow, recommend structural expansion before styling work.
